# Security Policy

Hypas Development takes responsible security reporting seriously.

## Reporting a vulnerability

Do not publish suspected vulnerabilities, bypasses, exploit payloads or licence issues in a public GitHub issue.

Report security concerns privately through:

- Discord: `https://discord.gg/TeBh6UfHEm`
- Email: `thelobby0626@gmail.com`

Include:

- A clear summary
- Affected version
- Reproduction steps
- Relevant logs or evidence
- Potential impact
- Whether the issue has been disclosed elsewhere

## Responsible disclosure

Please allow reasonable time for investigation and remediation before public disclosure.

Hypas Development may not provide detailed confirmation of detection logic, thresholds or protected implementation where doing so could weaken customer security.

## Scope

Valid reports may include:

- Authentication or permission bypass
- Licence or access-control weakness
- Remote event abuse affecting Hypas Anti-Cheat
- Evidence or audit-log integrity issues
- Unintended exposure of protected information
- Denial-of-service conditions caused directly by the resource

General cheat reports, false-positive reports and configuration support should be submitted through the standard support process.
