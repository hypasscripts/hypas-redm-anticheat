# Administration Panel

The Hypas Anti-Cheat panel centralises monitoring, investigation and response.

## Dashboard

The dashboard provides a high-level view of:

- Current server protection state
- Player risk
- Recent alerts
- Detection volume
- Health warnings
- Administrative activity

Use the dashboard to identify priorities, not to make a final decision from one number.

## Player view

The player view may include:

- Current risk level
- Strike history
- Detection timeline
- Aim or combat evidence
- Movement events
- Inventory or weapon concerns
- Staff notes
- Recent administrative actions

## Evidence viewer

The evidence viewer should be used to understand:

- What triggered the alert
- When it happened
- Which protection system reported it
- Whether the behaviour repeated
- What surrounding events occurred
- Whether legitimate server resources could explain it

## Administrative tools

Depending on configuration and version, authorised staff may have access to tools such as:

- Spectate
- Shadow Watch
- Refresh Snapshot
- Add Staff Note
- Freeze
- No Damage
- Inventory or Weapon Lock
- Isolate
- Reset Strikes
- Reset Aim Score
- Allow Teleport
- Return to Safe Position
- Remove Weapon

Use containment tools proportionately and document significant actions.

## Audit trail

Staff actions should be reviewable through the audit system.

A useful audit process records:

- Staff member
- Target player
- Action
- Time
- Reason
- Supporting evidence

## Recommended investigation workflow

1. Review the alert category.
2. Open the player's timeline.
3. Check repetition and severity.
4. Compare against known legitimate resources.
5. Observe the player where appropriate.
6. Add a staff note.
7. Apply containment only where justified.
8. Preserve evidence before punitive action.
