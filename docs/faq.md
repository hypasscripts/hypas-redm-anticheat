# Frequently Asked Questions

## Can Hypas Anti-Cheat stop every cheat?

No anti-cheat can honestly guarantee complete protection against every current and future attack.

Hypas Anti-Cheat uses layered prevention, monitoring, evidence and response systems to reduce risk and improve administrator visibility.

## Is the source code published here?

No.

This repository contains public documentation. The commercial resource and protected implementation are distributed through authorised channels.

## Which framework is supported?

Hypas Anti-Cheat is designed around RedM, VORP Core and oxmysql.

Custom frameworks or heavily modified systems may require additional integration.

## Will it automatically ban players?

Enforcement depends on configuration, mode and detection category.

Administrators should test settings carefully and understand which actions are automatic.

## Does it detect aimbot or ESP?

It includes combat and behavioural evidence systems intended to identify suspicious patterns and indicators.

Client-side cheats can be difficult to prove conclusively. Evidence should be reviewed in context.

## Will it cause false positives?

Any behavioural or rule-based security system can produce false positives when legitimate activity resembles abuse.

Hypas Anti-Cheat provides operating modes, evidence and tuning options to reduce this risk.

## Is it suitable for a live production server?

Yes, after correct installation, integration testing and configuration review.

Begin in Testing or Balanced mode rather than enabling the strictest enforcement immediately.

## Does it replace secure server-side coding?

No.

Sensitive economy, inventory, job and reward events should still validate requests server-side. Anti-cheat is one layer of a secure server.
