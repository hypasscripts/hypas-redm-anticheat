# Public Roadmap

This roadmap describes general development direction. It is not a guarantee of release dates or final implementation.

## Current priorities

- Stability and false-positive reduction
- Evidence quality
- Framework integration
- Performance monitoring
- Administrator usability
- Documentation
- Safe entity protection
- Event and resource hardening

## Under consideration

- Expanded investigation workflows
- Additional evidence correlation
- Improved reporting and export
- Broader framework adapters
- More granular staff permissions
- Enhanced operational health reporting
- Additional safe testing tools
- Public RedM server-security guidance

## Not published publicly

For customer security, the roadmap will not disclose:

- Exact detection signatures
- Honeypot names
- Threshold values
- Exploit-specific countermeasures
- Bypass research
